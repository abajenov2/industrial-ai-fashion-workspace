# Конфигурация: архитектор

Архитектор — резидент с расширенными правами и ответственностью за эталонную
связность системы.

Основные функции:

- собирать и версионировать эталон KA;
- удерживать онтологию, паспорта и связи `KA / DocKA / RL / TR / TRxTR / RLTR / A1.x`;
- проверять границы public / resident / private;
- после каждого согласованного пакета сначала собирать и проверять локальную
  версию на компьютере Архитектора;
- возвращать сомнительные места в эталонный источник, а не молча исправлять
  только GitHub-копию;
- выпускать согласованную версию открытого Свода для рабочих мест;
- не переносить расширенные права в чужие приватные данные без согласия владельца.
