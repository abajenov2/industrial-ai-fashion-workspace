# Эталонная марка: правила beta-паука

## Входные данные

Паук читает не «оценку человека о себе», а четыре вида следа:

```text
ответ в форме или рабочем месте
-> ссылка, таблица, документ или конкретный пример при наличии
-> состояние параметра
-> покрытие и уверенность вывода
```

Система не имеет права придумать факт, недостающую цифру, владельца процесса или
результат. Если материала недостаточно, результатом будет не оценка, а просьба
собрать ровно недостающий факт.

## Правило пропусков

| Статус ответа | Входит в зрелость зоны | Входит в покрытие | Что показывает человеку |
| --- | --- | --- | --- |
| `known` | да | да | рабочий факт или наблюдение |
| `unknown` | нет | нет | нужно собрать данные |
| `not_applicable` | нет | нет; исключается из знаменателя | вопрос не относится к этой марке сейчас |
| `deferred` | нет | нет | вернуться на следующем такте |

Если в зоне меньше двух известных применимых параметров или покрытие ниже 40%,
луч не рисуется числом. Вместо него отображается серый сегмент `недостаточно
данных`. Это запрещает ложную точность.

## Зрелость параметра

| Балл | Не техническое значение |
| ---: | --- |
| 0 | решения нет или оно держится только на памяти и случайных сообщениях |
| 1 | есть отдельные действия, цифры или файлы, но они не дают повторяемого решения |
| 2 | есть рабочий документ, владелец и ритм использования |
| 3 | решение связано с соседними зонами, измеряется и проверено повторяемым результатом |

Это beta-шкала. Ее нельзя использовать для сравнения марок между собой или для
автоматического присвоения статуса.

## Расчет зоны

Для каждой зоны берутся только ответы со статусом `known`:

```text
состояние зоны = средневзвешенное состояний применимых параметров
покрытие зоны = известные применимые параметры / все применимые параметры
уверенность зоны = минимум из качества доказательств ключевых параметров
```

Если одна зона содержит несколько уровней глубины, в beta сначала выбирается
минимальный набор по текущей цели, а не весь каталог вопросов. Поэтому итоговый
паук всегда показывает дату, версию вопросника и выбранную цель.

## Что делать после паука

1. Связать результаты с одной заявленной целью.
2. Выбрать не самый низкий луч, а разрыв, который сильнее всего мешает этой
   цели, дорого обходится и может быть проверен небольшим действием.
3. Предложить один следующий такт: нужный документ, сбор данных, эксперт,
   партнер, регулярная встреча или ограниченный пилот.
4. Назначить владельца, дату и наблюдаемый сигнал.
5. После действия пересчитать только затронутые параметры.

## Граница обмена

Полный паук остается `H.local`. Для `H.exchange` система формирует только
согласованную карточку:

```text
роль и стадия
-> одна цель
-> один разрыв
-> что уже готово
-> что предлагаю
-> кто нужен
-> формат первого действия
-> срок
```

Публичный слой возможен только как согласованный кейс после действия. В нем нет
диагностической карты, частных цифр, первичных ответов и внутренних документов.
