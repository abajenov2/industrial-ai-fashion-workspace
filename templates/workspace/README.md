# Рабочее место резидента Альянса

Это локальное рабочее место, связанное со Сводом знаний, платформой Альянса,
ролью резидента и его собственным цифровым следом.

Начало работы:

1. заполнить паспорт владельца;
2. отдельно указать контекст владельца и 1-3 текущие роли;
3. указать ближайшие 1-3 действия;
4. прочитать `workspace.policy.yaml` и правила доступа;
5. открыть `03_Библиотека_роли/01_Открытый_стандарт/Система_Альянса/navigation/START_HERE.md`;
6. пройти `03_Библиотека_роли/01_Открытый_стандарт/KA6_НАВИГАТОР.md`;
7. открыть `Свод_знаний_репозиторий/RLTR_НАВИГАТОР.md`;
8. открыть папку в Codex и проверить, что skill рабочего места доступен;
9. если ближайшая задача относится к марке, открыть
   `04_Проекты_и_рабочие_задачи/EM_v0_4_beta.md` и пройти только применимые
   блоки;
10. провести первую задачу по маршруту «вопрос -> KA6 -> KA -> статья -> роль ->
   эксперт/кейс -> действие с датой».

Это единое рабочее место. «Марка», «фабрика», «эксперт» и «Архитектор» не
являются четырьмя взаимоисключающими типами установки. Марка и фабрика описывают
организационный контекст, эксперт и Архитектор - роли. Ролей может быть много,
они добавляются без переустановки.

Не помещайте в GitHub пароли, токены, cookies, платежные данные, контактные базы,
чужие сырые транскрипты и документы с неясными правами.

Кейсы, TRxTR, профили и контакты открываются на платформе Альянса по ссылкам из
индексов. Общая библиотека не дублирует живой операционный контур сайта.

DocKA подключается выборочно: страница документа открывается на платформе, а
нужный файл скачивается из единого Yandex-каталога в приватную задачу владельца.
