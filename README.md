# Рабочее место резидента Альянса x Beinopen

Это установочный пакет единого локального рабочего места резидента Альянса.
Он помогает Codex понимать устройство Альянса, находить знания, роли, процессы,
экспертов и кейсы, а затем превращать вопрос в следующее действие с датой.

Сейчас опубликована предварительная версия для тестовых установок. Команда
Архитектора продолжает обновлять содержание; базовая версия `1.0.0` планируется
к 22 июля 2026 года.

## Что находится в репозитории

Репозиторий состоит из четырех связанных частей:

1. `knowledge-base/` - согласованный открытый Свод знаний KA и карточки-ссылки
   на резидентские материалы.
2. `alliance-system/` - навигация по системе: CJM, роли, сборки, правила поиска,
   доступы и протоколы.
3. `templates/workspace/` - единый каркас личного рабочего места.
4. `skills/alliance-resident-workspace/` - операционные правила для Codex.

Каждый общий объект по возможности имеет два представления:

- Markdown для человека;
- YAML или JSON для поиска, проверки связей и работы ИИ-помощника.

## Одна установка, много ролей

При установке не нужно выбирать тип `brand`, `factory`, `expert` или
`architect`. Рабочее место едино, а внутри него раздельно фиксируются:

- владелец: человек или организация;
- организационные контексты: марка, фабрика, вуз, медиа, ритейл и другие;
- одна или несколько текущих ролей;
- программы и кооперационные контуры;
- права доступа и основание полномочий.

Роль можно добавить или изменить без переустановки. Расширенные права, включая
полномочия роли Архитектора, не выдаются по названию роли и проверяются отдельно.

## Самый простой способ

Передайте своему Codex ссылку на репозиторий и напишите:

```text
Установи единое рабочее место резидента Альянса в новую пустую папку
C:\Alliance\workspace. После установки прочитай START_HERE.md, открой паспорт
и помоги заполнить только владельца, текущие роли, права доступа и ближайшую
реальную задачу.
```

Codex должен сначала клонировать репозиторий, затем запустить установщик. Личные
файлы не отправляются обратно в публичный GitHub.

## Установка: macOS, Linux или Git Bash

```bash
git clone https://github.com/abajenov2/industrial-ai-fashion-workspace.git
cd industrial-ai-fashion-workspace
bash install.sh /c/SITES2/alliance-workspace
```

На macOS или Linux путь может выглядеть так:

```bash
bash install.sh "$HOME/Alliance/workspace"
```

Папка назначения должна быть новой или пустой. Установщик не перезаписывает
существующие данные.

## Установка: Windows PowerShell

```powershell
git clone https://github.com/abajenov2/industrial-ai-fashion-workspace.git
Set-Location industrial-ai-fashion-workspace
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Target C:\SITES2\alliance-workspace
```

После установки откройте созданную папку в Codex и напишите:

```text
Прочитай AGENTS.md и помоги пройти первый запуск рабочего места.
```

## Как пользоваться системой

Начальная точка: [START_HERE](./alliance-system/navigation/START_HERE.md).

| Задача | Раздел |
|---|---|
| Понять содержание отраслевого вопроса | [Свод знаний KA](./knowledge-base/КАРТА_СВОДА.md) |
| Пройти процесс по шагам | [CJM и пути резидента](./alliance-system/journeys/README.md) |
| Понять, кто действует | [Роли RL](./alliance-system/roles/README.md) |
| Найти проверенную повторяемую конструкцию | [Сборки A1.x](./alliance-system/assemblies/README.md) |
| Найти источник, эксперта, кейс или документ | [Поиск и маршрутизация](./alliance-system/navigation/SEARCH_AND_ROUTING.md) |
| Проверить доступ и порядок действия | [Правила](./alliance-system/rules/README.md) и [протоколы](./alliance-system/protocols/README.md) |
| Понять формат добавления новых объектов | [Стандарт развития библиотеки](./alliance-system/AUTHORING_STANDARD.md) |

Статьи внутри каждой KA расположены в порядке учебника. Короткое имя вида
`003_KA1.2_post-3869.md` показывает место в иерархии, индекс KA и ID источника
на платформе.

## Обновление общего слоя

Сначала обновите локальную копию репозитория:

```bash
git pull
```

macOS, Linux или Git Bash:

```bash
bash update-library.sh /c/SITES2/alliance-workspace
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-library.ps1 -Target C:\SITES2\alliance-workspace
```

Обновляются Свод знаний и общая навигация Альянса. Паспорт, роли владельца,
задачи, выбранные документы и личный цифровой след не затрагиваются.

## Что публично, а что нет

В публичном репозитории находятся:

- открытые статьи Свода;
- карточки-ссылки на материалы, доступные только резидентам;
- общие CJM, форматы ролей, правила, протоколы и сборки;
- ссылки на профили, авторов, кейсы и источники на платформе Альянса.

В репозитории нет полных текстов закрытого DocKA, личных настроек, контактов,
сырых транскриптов, переписок, cookies, токенов и коммерческих данных
резидентов. GitHub не является хранилищем личного рабочего места.
