# Рабочее место резидента Альянса x Beinopen

Это установочный пакет для локального рабочего места марки, эксперта, фабрики
или Архитектора. Он помогает Codex понимать устройство Альянса, находить
материалы Свода знаний, авторов и связанные кейсы, а затем превращать вопрос в
следующее действие.

Сейчас опубликована предварительная версия для пилотных установок. Команда
Архитектора продолжает обновлять содержание; пользователю не нужно вручную
проверять весь Свод. Базовая версия `1.0.0` планируется к 22 июля 2026 года.

## Что происходит при установке

1. Репозиторий копируется с GitHub на ваш компьютер.
2. Установщик создает новое рабочее место выбранного типа.
3. В него добавляются паспорт, профиль роли, правила доступа, KA6-навигатор,
   открытый Свод знаний и skill для Codex.
4. Личные данные появляются только потом, когда владелец заполняет рабочее место.

Выберите тип:

- `brand` — марка;
- `expert` — эксперт;
- `factory` — фабрика;
- `architect` — роль Архитектора без личного архива и закрытых материалов.

## Самый простой способ

Откройте Codex и передайте ему ссылку на этот репозиторий вместе с просьбой:

```text
Установи рабочее место Альянса типа brand в новую пустую папку
C:\Alliance\brand-workspace. После установки открой паспорт и помоги заполнить
только минимальные поля для первого запуска.
```

Codex должен сначала склонировать репозиторий, затем запустить подходящий
установщик. Он не должен помещать личные файлы обратно в публичный репозиторий.

## Установка вручную: macOS, Linux или Git Bash

Сначала скачайте репозиторий и перейдите в его папку:

```bash
git clone https://github.com/abajenov2/industrial-ai-fashion-workspace.git
cd industrial-ai-fashion-workspace
```

Затем создайте рабочее место. Пример для марки в Git Bash на Windows:

```bash
bash install.sh brand /c/SITES2/brand-workspace
```

Примеры для других типов:

```bash
bash install.sh expert "$HOME/Alliance/expert-workspace"
bash install.sh factory "$HOME/Alliance/factory-workspace"
bash install.sh architect "$HOME/Alliance/architect-workspace"
```

Папка назначения должна быть новой или пустой: установщик не перезаписывает
существующие данные.

После каждого обновления GitHub автоматически проверяет установку всех четырех
типов рабочего места на Linux и Windows.

## Установка вручную: Windows PowerShell

```powershell
git clone https://github.com/abajenov2/industrial-ai-fashion-workspace.git
Set-Location industrial-ai-fashion-workspace
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Type brand -Target C:\SITES2\brand-workspace
```

Если нужен другой тип, замените `brand` на `expert`, `factory` или `architect`.

После установки откройте созданную папку в Codex и напишите:

```text
Прочитай AGENTS.md и помоги начать работу с паспорта рабочего места.
```

## Обновление общей библиотеки

Сначала обновите локальную копию репозитория:

```bash
git pull
```

macOS, Linux или Git Bash:

```bash
bash update-library.sh /c/SITES2/brand-workspace
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-library.ps1 -Target C:\SITES2\brand-workspace
```

Обновляется только общая библиотека. Паспорт, задачи, выбранные документы и
личный цифровой след владельца не затрагиваются.

## Что публично, а что нет

Этот GitHub-репозиторий публичный. В нем находятся только:

- открытые статьи Свода;
- карточки-ссылки на материалы, доступные только резидентам;
- общие шаблоны и правила;
- ссылки на профили, авторов и кейсы на платформе Альянса.

В репозитории нет полных текстов закрытых материалов, файлов DocKA, cookies,
токенов, контактных баз, приватных переписок, сырых транскриптов и коммерческих
данных резидентов. GitHub не является хранилищем личного рабочего места.
