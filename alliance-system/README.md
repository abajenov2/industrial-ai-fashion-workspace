# Система навигации рабочего места Альянса

Этот слой объясняет, как пользоваться не отдельным текстом, а всей системой
Альянса: знаниями, ролями, пошаговыми маршрутами, правилами, кейсами и
повторяемыми сборками.

Он не заменяет платформу Альянса. Платформа остается источником актуальных
публикаций, профилей, событий, комментариев и режимов доступа.

## Объекты системы

| Объект | На какой вопрос отвечает | Читаемый формат | Машинный формат |
|---|---|---|---|
| KA | что устроено и почему | статья / глава Markdown | каталог JSON |
| DocKA | каким документом работать | страница и описание | карточка документа |
| CJM | какими шагами пройти путь | пошаговый Markdown | YAML этапов |
| RL | кто действует сейчас | карточка роли | YAML роли |
| RLTR | как роль развивается | навигатор траектории | связи в индексе |
| TR | что реально сделал один владелец | локальный цифровой след | запись состояния |
| TRxTR | где пересеклись несколько траекторий | кейс на платформе | ссылочная карточка |
| A1.x | как повторяется проверенная сборка | паспорт сборки | YAML модулей и связей |
| правило / протокол | что разрешено и как координироваться | Markdown | ссылки и статусы |

## Главное правило

```text
запрос владельца
-> навигация
-> подходящий объект системы
-> источник истины
-> действие с владельцем и датой
-> цифровой след
-> уточнение роли, CJM, KA или сборки
```

Начните с [START_HERE](./navigation/START_HERE.md).

Формат развития: [стандарт добавления объектов](./AUTHORING_STANDARD.md).
План следующих волн: [ROADMAP](./ROADMAP.md).
