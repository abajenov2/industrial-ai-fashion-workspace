# Краткие правила поиска

1. Ищите не по одному слову, а по задаче, роли, стадии и ожидаемому результату.
2. Для определения используйте KA или глоссарий.
3. Для последовательности действий используйте CJM.
4. Для ответственности используйте RL/RLTR.
5. Для человека используйте индекс авторов и профиль Альянса.
6. Для рабочего документа используйте DocKA и проверяйте доступ.
7. Для совместного кейса используйте TRxTR.
8. Для повторяемой конструкции используйте только валидированную A1.x.
9. При конфликте GitHub и платформы проверяйте версию и источник на платформе.
10. Всегда завершайте поиск действием с владельцем и датой.
