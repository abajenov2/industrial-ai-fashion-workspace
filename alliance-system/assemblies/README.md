# Модульные сборки A1.x

A1.x описывает повторяемую сборку нескольких ролей, документов, знаний,
процедур и связей вокруг проверяемой задачи.

Единичный участник, встреча, событие, тема или кейс не является A1.x. Сначала
фиксируются TR и TRxTR. Кандидат сборки появляется, когда несколько похожих
кейсов показывают повторяемую конструкцию.

## Условия появления сборки

- потребности заинтересованных сторон;
- матрица требований;
- целевые показатели;
- ресурсные ограничения;
- библиотека типовых модулей;
- последовательность взаимодействия;
- несколько подтверждающих TRxTR;
- пилотная валидация;
- возможность повторения.

```text
матрица требований
+ типовые модули
+ уникальная конфигурация под контур
-> A1.x
```

Формат: [ASSEMBLY_CARD_FORMAT](./ASSEMBLY_CARD_FORMAT.md).
Источник рамки: [A1 на платформе Альянса](https://alliance.beinopen.ru/post/2169/).
