# Поиск и маршрутизация

## Порядок поиска

1. Определить объект запроса: знание, процесс, роль, документ, человек, кейс,
   правило или сборка.
2. Проверить локальный машинный индекс.
3. Открыть читаемый Markdown и понять контекст.
4. Перейти к источнику на платформе и проверить актуальность и доступ.
5. Вернуть не список ссылок, а короткий маршрут к действию.

## Приоритет источников

```text
платформа Альянса для опубликованного стандарта и доступа
-> согласованный GitHub release для установленной версии библиотеки
-> рабочий реестр для структурированных операционных состояний
-> локальное рабочее место для приватного контекста и задач
-> архив и сырье только как материал для обработки
```

## Правила ответа Codex

- «что это» - дать определение, KA и источник;
- «как сделать» - дать CJM, контрольные точки и дату следующего шага;
- «кто поможет» - дать основание рекомендации и профиль на платформе;
- «что скачать» - выбрать один нужный DocKA и проверить право доступа;
- «как повторить» - проверить, есть ли A1.x, а не называть сборкой единичный кейс;
- «что дальше» - сверить паспорт, TR и ближайшую дату.

Не выдавайте совпадение по ключевому слову за рекомендацию. Связь должна
подтверждаться авторством, ролью, кейсом, TRxTR или утвержденной сборкой.
