# Роли RL

Роль - конфигурация единого рабочего места, а не отдельный вид установки.

Роль описывается через:

- RBS - требования к результату;
- FBS - функции;
- SBS - инструменты, документы, люди и системы;
- WBS - работа и ритм;
- DSM - связи с KA, DocKA, CJM, TR, TRxTR, RLTR и A1.x.

Один владелец может иметь несколько ролей. Марка, фабрика, вуз или ритейлер
обычно являются организационными контекстами, а не ролями. Расширенные права не
выдаются по названию роли.

Формат карточки: [ROLE_CARD_FORMAT](./ROLE_CARD_FORMAT.md).
