# История версий

## 0.4.1-preview - 2026-07-14

- добавлен write-preflight `create -> write -> read -> delete` для Windows и
  OneDrive; реальное обновление выполняет его автоматически;
- dry-run получил явный флаг `-CheckWriteAccess`, который проверяет возможность
  staging без изменения действующей библиотеки;
- временные staging- и backup-имена сокращены, чтобы снизить риск ограничения
  длины пути в Windows PowerShell 5.1;
- копирование и перемещение каталогов переведено на буквальные .NET-операции,
  не интерпретирующие путь как шаблон;
- перед заменой проверяются `VERSION.json`, SHA-256 контрольного файла и
  совпадение количества файлов и каталогов;
- добавлен отдельный Windows-регрессионный сценарий для существующей библиотеки
  с `.keep`, кириллического OneDrive-подобного пути и безопасного отказа
  preflight в ограниченном контуре;
- исправление подготовлено по второму живому тесту рабочего места Ланы; ручное
  копирование при ошибке по-прежнему запрещено.

## 0.4.0-preview - 2026-07-14

- KA4 синхронизирован с обновленным эталоном из 56 опубликованных глав;
- добавлены резидентские карточки `KA4.2.6`, `KA4.2.7` и `KA4.2.8` со
  ссылками на `post/4032`, `post/4033` и `post/4034`;
- 13 открытых глав KA4 по-прежнему зеркалируются целиком, 43 резидентские главы
  KA4 представлены только карточками без текста;
- число будущих узлов KA4 уменьшилось с 10 до 7, два якорных раздела сохранены;
- для `post/4032` зафиксированы владелец Константин Кривошеин и соавтор Федор
  Шумилов, для `post/4034` Екатерина Непочатых;
- целевой автор `post/4033` Ольга Федотова оставлена без ссылки до нахождения
  точного аккаунта;
- `update-library` получил режимы `DryRun` и `LibraryOnly` для существующих
  рабочих мест;
- обновление общей библиотеки теперь сначала делает staging и валидацию новой
  копии, а затем только меняет действующую библиотеку;
- добавлены регрессионные сценарии для Windows PowerShell 5.1 с
  OneDrive-подобным кириллическим путем и старым минимальным рабочим местом;
- README и Протокол разделяют новую установку, обновление библиотеки,
  изменение ролей и обновление skill;
- импортер проверяет числа по manifest и умеет подключать отдельный журнал
  последующей сайтовой волны;
- clean-export audit и четыре локальных сценария установки пройдены.

## 0.3.1-preview - 2026-07-14

- четыре взаимоисключающих типа установки заменены единым рабочим местом
  резидента;
- владелец, организационный контекст, роли, программы и права доступа теперь
  фиксируются как разные оси;
- добавлен общий слой `alliance-system` с точкой входа, правилами поиска,
  CJM, ролями, сборками и протоколами;
- добавлены три начальных CJM: первый запуск, рабочий цикл от вопроса к
  результату, встреча и цифровой след;
- для CJM, ролей, сборок и навигации добавлены машиночитаемые YAML-индексы и
  схемы;
- `update-library` обновляет Свод знаний и общую систему навигации, не меняя
  личный слой владельца;
- 40 материалов KA4 с режимом «Только для своих» по-прежнему представлены в
  публичном GitHub только карточками-ссылками;
- автоматические проверки моделируют человека, организацию, несколько ролей и
  границу расширенных полномочий на Linux и Windows.

## 0.3.0-preview - 2026-07-14

- выполнен повторный импорт KA4 из проверенного эталона Архитектора;
- в учебник включены 53 канонические главы KA4 в структурном порядке;
- 13 публичных глав зеркалируются полным текстом, 40 резидентских материалов
  представлены в открытом GitHub только карточками-ссылками;
- 10 будущих узлов и 2 якорных раздела добавлены в оглавление и машинный
  каталог без создания пустых статей;
- подтвержденное авторство перенесено для 36 экспертных владельцев; 17
  нерешенных атрибуций оставлены без догадок;
- добавлен воспроизводимый импорт KA4 с машиночитаемым diff и проверками
  доступа;
- DocKA4 не включен и остается отдельной следующей волной.

## 0.2.0-preview - 2026-07-13

- статьи внутри каждой KA расположены по порядку учебника, а не по ID поста;
- добавлены короткие ASCII-имена статей и оглавления KA1-KA6;
- машинный каталог получил `primary_ka`, `parent_ka`, `hierarchy_depth`,
  `outline_order` и `node_type`;
- закрытые материалы остаются только карточками, а приватные пути удалены из
  публичного машинного каталога;
- README переписан как инструкция первого запуска;
- команды установки начинаются с `git clone` и `cd`;
- добавлены установщик и обновление библиотеки для Windows PowerShell;
- добавлена автоматическая проверка прежних четырех тестовых конфигураций на
  Linux и Windows.

Содержание Свода использует тот же предварительный снимок, что и версия
`0.1.0-preview`. Повторный импорт из эталона Архитектора не выполнялся.

Первая публичная предварительная сборка общего рабочего места и открытой копии
Свода знаний.
